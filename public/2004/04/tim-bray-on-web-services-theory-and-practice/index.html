Tim Bray on Web Services Theory and Practice

Stefan Tilkov, Apr 28, 2004

Not necessarily agreeing with it, still worth linking to: Tim Bray about Web Services. What I really do like:

[A]pplications that try to abstract away the fact that they’re exchanging XML messages will suffer for it. Thus, I tend to consider any “Web Services” specification that’s more than two steps away from XML for content and URIs for addressing to be somewhat suspect.

What I assume will create an outcry among Workflow and BPEL people:
