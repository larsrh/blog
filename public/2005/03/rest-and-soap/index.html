REST and SOAP

Stefan Tilkov, Mar 12, 2005

Savas Parastatidis builds part of the Web on SOAP and concludes:

Both the REST and WS communities have a lot to learn from each other. We should constructively educate one-another and learn from what ‘the other side’ has to say instead of declaring each other’s demise.

+1.

I thought about creating a list of links about the current debate, but I would not be able to add much to Tim Bray’s post.

(BTW, to cut off some predictable comments: No, I don’t believe we should build another Web on top of the existing one; I don’t think Savas was that serious, either.)
